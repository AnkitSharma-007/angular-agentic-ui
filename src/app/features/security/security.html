Headline guarantee
cloud_off

Pure frontend. No backend.

The app runs entirely in your browser and calls {{ config.geminiEndpoint }} directly using the key you provide. There is no server in the middle to log, leak, or invoice.

vpn_key

Where your key lives

Two storage modes, both local to your device.

  • Default
    Session-only. Held in memory and sessionStorage for the active tab. Wiped when you close the tab.
  • Opt-in
    Encrypted on this device. Encrypted with a passphrase via WebCrypto AES-GCM (PBKDF2-derived key, SHA-256, 250 000 iterations) and stored in localStorage. The passphrase never leaves your browser.
shield

Where your key is never sent

The outbound traffic budget is short.

  • check_circle No backend, so no transit to a server we control.
  • check_circle No third-party analytics, logging, or telemetry endpoints.
  • check_circle No third-party scripts at runtime, beyond Google Fonts.
perm_media

Images, voice & saved runs

Media stays on the same short path as everything else.

  • check_circle Images you attach are downscaled in your browser and sent inline to {{ config.geminiEndpoint }} — never uploaded to any other host.
  • check_circle Voice input uses your browser's built-in speech recognition. Depending on your browser, that audio may be processed by its speech provider (often cloud-based) — but this app itself never records, stores, or sends your audio.
  • check_circle Saved runs (including any attached media) live only in this browser's IndexedDB so replays stay self-contained.
auto_awesome

Agent-authored tools

Self-extension, always under your control.

  • check_circle When the agent proposes a new tool, it pauses for your review — nothing is registered until you approve it. You can edit or reject any proposal.
  • check_circle Synthesized tools only return the JSON template you approve; they never call real services or run arbitrary code.
  • check_circle Approved tools are stored only in this browser's IndexedDB and are labelled Agent in the Tools library so their origin is always clear.
